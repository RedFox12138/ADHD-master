# 快速测试说明

如果您暂时没有音频文件，可以先使用以下方法测试：

## 方案一：使用在线音效库（推荐用于测试）

可以临时使用在线音效URL进行测试。修改 `audioManager.js` 中的文件路径：

```javascript
// 将本地路径
this.soundEffects[effect].src = `/audio/${effect}.mp3`;

// 临时改为在线URL（示例）
this.soundEffects[effect].src = `https://example.com/sounds/${effect}.mp3`;
```

## 方案二：静音模式测试

默认情况下，即使没有音频文件，游戏也能正常运行，只是没有声音。
您可以点击顶部工具栏的🔊图标来关闭音效，避免加载错误。

## 方案三：生成简单测试音频

可以使用以下免费工具快速生成测试音效：

1. **Audacity** (免费开源)
   - 生成 → 音调 → 选择频率和时长
   - 导出为MP3格式

2. **在线工具**
   - https://www.beepbox.co/ - 8bit音效生成器
   - https://sfxr.me/ - 游戏音效生成器

## 音效文件命名规范

确保文件名与代码中定义的完全一致：

- ✅ `button_click.mp3`
- ✅ `turret_shoot.mp3`
- ❌ `button-click.mp3` (错误：使用了连字符)
- ❌ `ButtonClick.mp3` (错误：大小写不匹配)

## BGM添加步骤

1. 准备两个MP3文件：
   - `main_bgm.mp3` - 主界面音乐
   - `game_bgm.mp3` - 游戏中音乐

2. 放入 `/audio/` 目录

3. 在 `index.js` 中取消注释：
   ```javascript
   // 第147行左右 - 主界面BGM
   audioManager.playBGM('main_bgm', 'ADHD训练 - 主界面');
   
   // 第721行左右 - 游戏BGM
   audioManager.playBGM('game_bgm', 'ADHD训练 - 游戏中');
   ```

## 注意事项

- 音频文件会增加小程序包体积，建议每个文件 < 500KB
- 测试时可以先关闭音效，等有了合适的音频再开启
- 音频加载失败不会影响游戏正常运行
